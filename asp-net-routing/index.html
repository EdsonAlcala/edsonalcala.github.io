ASP.NET Routing

Con ASP.NET routing podemos utilizar URLs que no se referencien a archivos especificos en un sitio web.

Tipicamente en una aplicación de ASP.NET cuando realizamos una petición a una URL especifica, está URL está mapeada a un archivo físico como por ejemplo
http://server/application/Products.aspx?id=4 se refiere aun archivo que es llamado Products.aspx, notese además que estamos pasando el valor de id=4 a traves del query string.

En ASP.NET routing nosotros podemos definir un patrón de URL que contenga placeholders para valores que son usados cuando manejamos peticiones a URL.

En tiempo de ejecución, las piezas de la URL que siguen a la aplicación son convertidas en valores discretos basado en un patrón de URL que hemos definido.

Para agregar rutas debemos hacerlo agregandolas a la propiedad estatic Routes de la clase RouteTable.

protected void Application_Start(object sender, EventArgs e)
{ RegisterRoutes(RouteTable.Routes); }

public static void RegisterRoutes(RouteCollection routes)
{ routes.Add(new Route ( "Category/{action}/{categoryName}" , new CategoryRouteHandler() )); }
